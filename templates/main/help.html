Help for New POD/Data Center

Antigua help information

This is what you are building

Assumptions

The wiring of the spine/leaf architecture follows a standard deploymet. All spine01 connections go to port 1 of the leaf(s). Then the next spine (spine02), will have its interfaces connected to port 2 of all leaves and so on until the fabric is complete.

The names of the Arista Configlets follows the format of (POD name).(switch_role). EX. pod2.spine01 The hostnames, on the devices themselves, the host name MUST match the configlet name. EX pod2.spine01. ANY CHANGE TO THIS WILL CAUSE THIS APPLICATION TO FAIL!!!!!!

Antigua is a mongo database used for managing variables for spine/leaf networks. The fields in the database hold the minimal variables needed to spawn all the information needed be managed by ansible. After performing the Zero Thouch operation, the switches will be ready for ansible-playbooks.

Using a zero touch provisioning techops process with a Linux host configured with DHCP option 67 truly provides for no touch (of the switch!).
Arista switches load a python file and execute it locally and create the basic configuration necessary for Ansible management.
Once the ZTP process has ran, the switches are ready to be added to the Cloud Vision Portal server or be managed by the playbooks used with this program.
Current Behavior
This application uses "SEED" variables. You enter the seed management IP address and the system will generate all the IP addresses required for the management interfaces. Same with loopback addresses and Router Id's. At this release the primary fileds are as follows.
POD

The name of the POD. Used to track different data center installations. (POD1, DC2-POD2, etc)


Management IP

Seed IP for all of the management addresses assigne to the OOB management port


Networks/Fabric

The seed address used to populate the L3/Fabric connections between switches


Router ID

Seed address for router id's


Gateway

The default gateway for the O.O.B. management network


Loopbacks

The seed IP for the switch loopback addresses


SSH User

Name of the management user. Like "admin"


SSH Password

Password of the SSH User


Spines

Dropdown for the total number of spines


Leafs

Dropdown for the total number of leafs
